# OctoAcme — Project Management Docs

This folder contains OctoAcme's project management process documents. The goal is to give new and current team members a single entry point to discover how we initiate, plan, execute, release, and continuously improve work.

## Quick overview

We follow an iterative, customer-centric approach with defined roles, checklists, and artifacts to reduce single-person dependencies and make project work repeatable and discoverable. Key phases:

- Initiation — define the problem, stakeholders, success metrics, and go/no-go decision.
- Planning — break work into shippable increments, estimate, capture dependencies & risks.
- Execution & Tracking — day-to-day delivery rhythms, PR/CI practices, demos, and metrics.
- Release & Deployment — pre-release checks, deployment checklist, rollback and incident playbook.
- Risk & Communication — maintain a risk register, stakeholder updates, escalation paths.
- Retrospective & Continuous Improvement — capture learnings and track action items.
- Roles & Personas — responsibilities for PM, PdM, Developers, QA and other stakeholders.

## How to use these docs
- Use the One-pager and project README as the single source of truth for project decisions.
- Follow the checklists in each doc for initiation, planning, releases, and retrospectives.
- Add process-specific files into `.copilot/` if you want Copilot Spaces to use them as context (per the project guidance).
